### Scenario: Setting a 2.5 Second Wait

```

???????????????????????????????????

?                 ?? cursor here   ?
???????????????????????????????????

Step 3: User must:
  a) Select all text (Ctrl+A)
  b) Type "2.5"
```

## Flexibility Comparison

### Supported Input Formats

**OLD Method:**
```
Accepts ONLY:
- Decimal numbers (e.g., 2.5)
- Must be in seconds
- No units shown or accepted
```

**NEW Method:**
```
Accepts ALL of these:
? "2.5"         (implicit seconds)
? "2.5s"        (explicit seconds)
? "2.5 seconds" (verbose)
? "2 sec"       (abbreviated)
? "500ms"       (milliseconds!)
? "1000 ms"     (ms with space)
? "0.5"         (half second)

= Same result: 2.5 seconds
```

## Learning Curve

### OLD: Number Input

```
User Questions:
1. "Is this seconds or milliseconds?"
   ? Must check documentation
   
2. "How do I set 500ms?"
   ? Must mentally calculate: 500ms = 0.5s
   
3. "Why won't the spinner work?"
   ? May be too small, or browser issue
   
4. "What's the maximum value?"
   ? No indication, must try and fail

Learning Time: 5-10 minutes
User Confidence: Low (trial and error)
```

### NEW: Text Input

```
User Questions:
1. "How do I edit this?"
   ? "Click to edit" hint shown
   
2. "What format should I use?"
   ? Placeholder: "e.g., 2.5s"
   ? Accepts multiple formats
   
3. "Is 500ms okay?"
   ? ? Works perfectly!
   
4. "What if I enter too much?"
   ? Auto-corrects with clear message

Learning Time: 30 seconds
User Confidence: High (instant feedback)
```
