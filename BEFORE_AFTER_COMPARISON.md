## Interaction Patterns

### BEFORE - Hover-Based (Desktop)
```css
/* Doesn't work well on mobile */
.button:hover {
  background: darker;
}

/* User must hover (impossible on touch) */
```

### AFTER - Touch-Based (Mobile)
```css
/* Perfect for mobile */
.button:active {
  background: darker;
}

/* Immediate visual feedback */
touch-manipulation
```

## CSS Specificity

### BEFORE
```css
/* Minimal mobile support */
.min-h-screen         /* Viewport issues */
.max-w-6xl           /* Whitespace */
.py-4 .px-4          /* Too much padding */
.hover:bg-gray-200   /* Doesn't work on touch */
```

### AFTER
```css
/* Mobile-first */
.h-screen            /* Fixed viewport */
.overflow-hidden     /* Prevents scroll issues */
.safe-top           /* iPhone notch support */
.safe-bottom        /* Home indicator support */
.touch-manipulation /* Better touch response */
.active:bg-gray-200 /* Works on touch */
.min-h-[44px]       /* iOS minimum */
.min-h-[48px]       /* Android guideline */
```
